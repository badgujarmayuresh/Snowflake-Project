## Context

**Key finding:** Cortex Agent REST API is blocked in SiS **warehouse runtime** — but `SNOWFLAKE.CORTEX.DATA_AGENT_RUN` is a SQL function that wraps the same API and works from `session.sql()` in warehouse runtime. This lets us call `SMART_BI_AGENT` without a container runtime or compute pool — no extra cost.

---

## Implementation Steps

### Step 1 — Create `streamlit/streamlit_app.py` (local)
Single-file Streamlit app with:
- Chat interface (`st.chat_message`) with full message history
- Calls agent via `session.sql("SELECT SNOWFLAKE.CORTEX.DATA_AGENT_RUN(...)")` 
- Parses JSON response to extract assistant text
- Sidebar with example questions + project info
- Handles thinking/tool events (shows them in an expander)

### Step 2 — Create `sql/07_streamlit/01_deploy_streamlit.sql` (local)
SQL deploy script that:
1. Creates an internal stage `STGINT_STREAMLIT` in `APP_CORTEX`
2. Uploads `streamlit_app.py` via `PUT` (run locally from SnowSQL/Snowsight)
3. Creates Streamlit object pointing to the stage

```sql
-- Create stage
CREATE STAGE IF NOT EXISTS DB_DEMO_MAYURESH.APP_CORTEX.STGINT_STREAMLIT;

-- PUT command (run from SnowSQL locally)
-- PUT file://streamlit/streamlit_app.py @DB_DEMO_MAYURESH.APP_CORTEX.STGINT_STREAMLIT AUTO_COMPRESS=FALSE;

-- Create Streamlit app
CREATE OR REPLACE STREAMLIT DB_DEMO_MAYURESH.APP_CORTEX.SMART_BI_APP
FROM '@DB_DEMO_MAYURESH.APP_CORTEX.STGINT_STREAMLIT'
MAIN_FILE = 'streamlit_app.py'
QUERY_WAREHOUSE = COMPUTE_WH
TITLE = 'Smart BI Agent';

ALTER STREAMLIT DB_DEMO_MAYURESH.APP_CORTEX.SMART_BI_APP ADD LIVE VERSION FROM LAST;
```

### Step 3 — Create Streamlit stage + deploy via Snowsight upload
- Create the stage via SQL
- Upload `streamlit_app.py` via Snowsight UI file upload (no SnowSQL needed)
- Create + activate the Streamlit object

### Step 4 — Create `.cortex/skills/streamlit-smartbi.md` (local)
Skill documenting the app architecture, `DATA_AGENT_RUN` pattern, deployment steps, and response parsing.

### Step 5 — Verify
- `SHOW STREAMLITS IN DB_DEMO_MAYURESH.APP_CORTEX` confirms the app object exists
- Visit app URL in Snowsight to confirm it loads
