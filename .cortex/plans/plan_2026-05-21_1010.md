## Context

**Explored:**
- Architecture.md defines Cortex Search as indexing "key business insights, report summaries, anomaly descriptions" for semantic search
- We have `TBL_OLIST_ORDER_REVIEWS` (7,000 reviews, ~6,000 with text) — ideal for semantic search on customer feedback
- We have `TBL_MFG_MACHINE_DOWNTIME` (500 records with REASON text) — good for searching operational incidents
- Cortex Search Service requires: `ON <search_column>`, `ATTRIBUTES` (filterable columns), `WAREHOUSE`, `TARGET_LAG`, and a source query

**Approach:** Create a Cortex Search service that indexes **customer reviews** enriched with order/product context, enabling business users to semantically search through customer feedback. Also create a second service for **manufacturing incidents** (downtime + defects). This aligns with the architecture's stated purpose.

---

## Implementation Steps

### Step 1: Create a search-ready table for customer insights
Create `APP_CORTEX.TBL_SEARCH_CUSTOMER_INSIGHTS` that combines review text with order/product context into a single searchable text column, plus filterable attributes (review_score, product_category, customer_state).

### Step 2: Create a search-ready table for manufacturing incidents
Create `APP_CORTEX.TBL_SEARCH_MFG_INCIDENTS` that combines downtime reasons and defect descriptions with machine/plant context into a searchable text column, plus filterable attributes (incident_type, plant_location, machine_type).

### Step 3: Create Cortex Search Service for customer insights
```sql
CREATE CORTEX SEARCH SERVICE DB_DEMO_MAYURESH.APP_CORTEX.SVC_SEARCH_CUSTOMER_INSIGHTS
ON SEARCH_TEXT
ATTRIBUTES REVIEW_SCORE, PRODUCT_CATEGORY, CUSTOMER_STATE
WAREHOUSE = COMPUTE_WH
TARGET_LAG = '1 hour'
AS (SELECT ... FROM APP_CORTEX.TBL_SEARCH_CUSTOMER_INSIGHTS);
```

### Step 4: Create Cortex Search Service for manufacturing incidents
```sql
CREATE CORTEX SEARCH SERVICE DB_DEMO_MAYURESH.APP_CORTEX.SVC_SEARCH_MFG_INCIDENTS
ON SEARCH_TEXT
ATTRIBUTES INCIDENT_TYPE, PLANT_LOCATION, MACHINE_TYPE
WAREHOUSE = COMPUTE_WH
TARGET_LAG = '1 hour'
AS (SELECT ... FROM APP_CORTEX.TBL_SEARCH_MFG_INCIDENTS);
```
